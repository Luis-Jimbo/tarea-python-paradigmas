## Descripción de los Programas
1. **`programacion_tradicional/`**: Solución estructurada mediante funciones y variables. Solicita datos usando el teclado (`input()`).
2. **`programacion_poo/`**: Solución modular dividida en dos archivos. Define una clase `Mascota` (en `mascota.py`) y gestiona los objetos en `main.py`.

---

## Reflexión: Diferencias entre la Programación Tradicional y la POO
* **Organización:** En el enfoque tradicional los datos y las funciones están separados. En POO, se agrupan dentro de una misma entidad (el objeto), imitando al mundo real.
* **Escalabilidad:** Con POO es mucho más fácil crear múltiples mascotas de manera ordenada creando instancias, mientras que en la tradicional requeriría colecciones de datos más complejas de mantener.
